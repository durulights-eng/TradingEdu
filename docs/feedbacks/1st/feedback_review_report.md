## 3. 안드로이드 앱 피드백 추가 검토 및 조치 가이드 (2026.07.01)

사용자 피드백으로 추가 인지된 안드로이드 기기 대응 이슈 및 버그에 대한 정밀 진단 결과와 수정 권장 코드 조치 방안입니다.

### 3.1 알림 토글 & 알림 아이콘
*   **피드백 사항**: 설치 시 권한을 최초 요청하고 거부해도 정상 진입되게 하되, 인앱 설정 탭에서 활성화 시 권한을 재차 물어보고, 사용자가 원하는 알림 시각(몇 시 몇 분)을 커스텀 지정하도록 지원. 알림 아이콘이 보이지 않거나 단색 사각형으로 깨지는 문제 해결.
*   **진단 결과**:
    1.  현재 `SettingsTab.tsx` 내의 알림 생성(`scheduleDailyReminder`)은 매일 오후 9시(`21:00`)로 고정 하드코딩되어 있습니다.
    2.  안드로이드 알림 아이콘의 경우, 리소스가 기본 앱 아이콘으로 매핑되어 있거나 투명도(Alpha)가 없는 컬러 이미지일 경우 안드로이드 OS 정책에 의해 하얀색 사각형으로 렌더링됩니다.
*   **수정 가이드**:
    *   **앱 시작 시 최초 권한 요청**: `src/App.tsx`의 마운트 시점(`useEffect`)에 `LocalNotifications.requestPermissions()`를 비동기로 호출하되, 결과에 상관없이 앱이 대시보드로 무사 진입되도록 처리합니다.
    *   **커스텀 알림 시간 설정 UI**: `SettingsTab.tsx`에서 알림 토글 스위치 활성화 시 사용자가 시간을 고를 수 있도록 시간 선택 컴포넌트(예: `<input type="time" />` 또는 시간/분 Select Box)를 렌더링하고, 설정한 시각은 `localStorage`(`chartmon_reminder_hour`, `chartmon_reminder_minute`)에 보관합니다.
    *   **알림 등록 스케줄링**: `LocalNotifications.schedule`을 호출할 때 하드코딩된 `21:00`을 사용자가 선택한 시/분 값으로 가변 지정합니다.
    *   **알림 아이콘 깨짐 해결**: 투명 바탕에 하얀색 실루엣만 가지는 `ic_stat_notification.png` 아이콘 리소스를 안드로이드 네이티브 경로(`android/app/src/main/res/drawable/`)에 추가하고, `capacitor.config.ts`의 LocalNotifications 플러그인 설정 및 schedule 시 `smallIcon: "ic_stat_notification"` 지정을 수행해야 합니다.

### 3.2 백그라운드 재진입 시 무한 로딩 이슈
*   **피드백 사항**: 앱을 백그라운드로 보냈다가 재진입 시 로딩 서클에서 화면이 멈추는 현상이 가끔 발생함 (강제 종료 후 재진입 시에는 무사히 로딩됨).
*   **진단 결과**: 
    1.  `src/App.tsx`는 구동 시 `isProfileLoading` 상태가 `isSupabaseConfigured(true)`로 초기화되어 프로필 동기화 완료 전까지 로딩 화면을 강제 렌더링합니다.
    2.  모바일 환경에서 네트워크 세션이 유실된 상태로 백그라운드에서 깨어날 때(Resume), Supabase가 세션 토큰을 리프레시하기 위해 던지는 fetch 요청에 명시적인 네트워크 타임아웃이 없어 대기 상태로 무한 대기(Pending)하게 됩니다. 이로 인해 `onAuthStateChange` 콜백이 트리거되지 않아 `isProfileLoading`이 영원히 `true`로 갇히게 됩니다.
*   **수정 가이드**:
    *   앱 구동 시, 세션 로딩 및 프로필 수집이 지연되는 경우 강제로 화면 진입을 허용하는 **안전 장치 타임아웃(Safety Timeout)**을 추가합니다.
    *   예를 들어, `App.tsx`의 마운트 시점에 4초 타이머를 작동시켜 시간 초과 시 무조건 `setIsProfileLoading(false)`로 설정해 게스트 캐시 데이터로 대시보드 진입을 우회 처리하게 만듭니다.

### 3.4 스트릭(Streak) 초기화 버그 (5일 고정 및 1일 리셋 불가)
*   **피드백 사항**: 앱 최초 진입 시 스트릭이 5일로 표기되고, 연속 학습을 실패해도 1일로 정상 리셋되지 않는 것으로 보임.
*   **진단 결과**:
    1.  `src/App.tsx`에서 스트릭 초기 상태 로드 시 삼항 연산자가 아닌 논리합(`||`) 연산자로 디폴트값을 매핑하고 있습니다:
        `Number(localStorage.getItem('chartmon_streak')) || 5`
    2.  이로 인해 신규 사용자의 로컬 스토리지에 값이 없을 경우(`null` ➡️ `0`으로 파싱) `5`로 fallback 되며, 학습을 놓쳐 스트릭이 `0`으로 리셋되는 경우에도 `0 || 5`가 연산되어 항상 `5`로 강제 복구되는 오동작을 보입니다.
*   **수정 가이드**:
    *   로컬 스토리지를 로드할 때 키 존재 여부를 엄격히 확인하도록 변경합니다:
        ```typescript
        const saved = localStorage.getItem('chartmon_streak');
        const initialStreak = saved !== null ? Number(saved) : 0; // 초기 설치 시 0일 혹은 첫 활성 1일로 시작
        ```

### 3.5 구글 로그인 다른 계정 전환 기능 미동작
*   **피드백 사항**: 소셜 로그인 시 기존 가입 계정으로만 강제 자동 로그인이 진행되어 다른 구글 계정을 선택해 가입할 수 없음.
*   **진단 결과**: Supabase `signInWithOAuth` 호출 시, 안드로이드 인앱 웹뷰/브라우저가 기존 Google 로그인 쿠키 세션을 메모리에 유지하기 때문에 계정 확인창 없이 즉시 자동 승인이 일어납니다.
*   **수정 가이드**:
    *   OAuth 요청 시 사용자에게 계정 선택 창을 명시적으로 보여주도록 Google 전용 쿼리 매개변수(`prompt=select_account`)를 강제 삽입하도록 코드를 변경합니다.
    *   `App.tsx` 내 `supabase.auth.signInWithOAuth` 호출 코드에 옵션을 주입합니다:
        ```typescript
        const { error } = await supabase.auth.signInWithOAuth({
          provider: 'google',
          options: {
            redirectTo,
            queryParams: {
              prompt: 'select_account'
            }
          }
        });
        ```
