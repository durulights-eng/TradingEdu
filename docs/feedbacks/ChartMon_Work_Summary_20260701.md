# ChartMon 프로젝트 작업 내역서 (Handoff Document)
**작성일: 2026년 7월 1일**

본 문서는 다음 코딩 에이전트가 ChartMon 프로젝트의 현재 상태와 직전 작업 내역을 완벽히 이해하고 이어서 작업할 수 있도록 작성된 인수인계 문서입니다.

## 1. 개요 및 최근 주요 수정 사항
최근 세션에서는 퀴즈 앱의 **차트 시각화 엔진(`ChartVisualizer.tsx`) 고도화** 및 **문제 DB 결함(가상 선 데이터 누락 및 텍스트 하드코딩) 수정**에 집중했습니다. 또한, 저작권 및 규제 문제를 피하기 위해 실존 기업명들을 모두 가명 처리했습니다.

## 2. 세부 작업 내역

### [1] 차트 엔진 (`src/components/ChartVisualizer.tsx`) 시각화 고도화
* **X축 날짜 라벨 출력 로직 개선**:
  * 캔들 개수가 적은 차트(8개 이하)에서 날짜 라벨이 누락되지 않고 모두 표시되도록 수정.
  * 긴 차트의 경우 라벨이 겹치지 않도록 동적으로 건너뛰는(skip) 로직 적용 완료.
* **추세선 색상(Color Contrast) 개선**:
  * 기존 파란색/빨간색 캔들과 추세선 색상이 겹쳐 가독성이 떨어지던 문제 해결.
  * 하락 추세선(`하락`): 쨍한 **밝은 청록색(Cyan, `#22d3ee`)**으로 변경.
  * 상승 추세선(`상승`): 쨍한 **밝은 녹색(Green, `#10b981`)**으로 변경.
* **콜아웃(Callout) 주석(Annotation) UI 도입**:
  * 기존에는 추세선 바로 옆에 텍스트를 렌더링하여 캔들과 글씨가 겹치는 문제 발생.
  * 트레이딩뷰 스타일의 콜아웃 방식 적용: 선의 끝점에서 차트 상단 빈 공간(Top Margin)으로 **얇은 점선 가이드라인**을 뽑아올린 후, 상단에 선의 명칭(예: "EMA 50")을 렌더링.
  * 여러 개의 선이 있을 경우(예: 볼린저 밴드, 다중 이평선), 글씨가 겹치지 않도록 인덱스에 따라 X축 위치를 엇갈리게(Stagger) 배치.
* **수평선(Horizontal Line) 스타일 변경**:
  * 저항선/지지선 등 수평선 본체는 점선(`dashed`)에서 확실한 **실선(`solid`)**으로 변경하여 직관성 확보. (연결선/가이드라인만 점선 사용)

### [2] 문제 DB (`src/data/quizzes/*.ts`, `*.js`) 데이터 클렌징 및 패치
* **가상 선 데이터(Drawings) 일괄 주입**:
  * "이평선 데드크로스", "볼린저 밴드 스퀴즈", "200일선 중첩" 등의 상황을 묘사하는 문제들(ID 25, 55, 69, 72 및 그 파생 변형 문제들)에 실제로 차트 위에 그려질 선 좌표(`drawings`) 데이터가 누락되어 있던 크리티컬 이슈 발견.
  * Node 스크립트를 통해 상황에 맞는 이평선/수평선/볼린저 밴드 곡선 좌표(Points) 데이터를 총 **84개 문제에 일괄 주입 완료**.
* **하드코딩된 라벨 텍스트 동기화 오류 수정**:
  * 퀴즈 난이도에 따라 차트의 스케일(Y축 가격)이 변형되는데, 특정 문제들에서 `저항선 (105)`와 같이 이전 기준 가격이 라벨 문자열에 하드코딩되어 있던 문제 발견.
  * 우측 눈금 가격과 텍스트가 불일치하는 혼란을 막기 위해, 모든 문제에서 `(105)` 같은 고정된 숫자를 정규식 스크립트로 삭제하고 **`저항선`**으로 깔끔하게 통일.
* **실존 기업명 가명 처리 (Pseudonymization)**:
  * 테슬라, 애플, 삼성 등 실제 기업명이 사용된 문제들을 모두 T모빌리티, A테크, S전자 등 한국식 가명(힌트형 가명)으로 일괄 치환 완료. (`fix_names.cjs` 스크립트 사용)

### [3] 빌드 및 배포 환경 (Android)
* **표준 배포 파이프라인 확립**:
  * React 소스코드 변경 시 `npm run android:sync`를 통해 Capacitor에 동기화.
  * `./gradlew assembleDebug`로 APK 빌드 후 `adb`를 이용해 에뮬레이터(`emulator-5556`)에 즉시 설치 및 자동 실행시키는 테스트 환경 구축 완료.

## 3. 다음 에이전트를 위한 제언 (Next Steps)
* **보조지표 패널 추가 고려**:
  * 현재 `ChartVisualizer.tsx`는 가격 캔들과 거래량(하단 25% 영역)만 렌더링 가능함. 
  * "상승 다이버전스(RSI)"나 MACD 관련 문제가 일부 존재하므로, 추후 하단에 독립된 **RSI/MACD 전용 보조지표 패널**을 그리는 레이아웃 업데이트가 필요할 수 있음.
* **에뮬레이터 반응형(Responsive) UI**:
  * 태블릿 지원을 고려하여 `ChartVisualizer.tsx` 내의 마진(Margin) 값이나 글씨 크기가 화면 너비(Width)에 따라 유연하게 커지는지(Fluid Typography) 추가 점검 요망.
