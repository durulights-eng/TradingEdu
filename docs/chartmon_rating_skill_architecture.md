# ChartMon Rating, Theory, and Skill Architecture

## 1. 레이팅 시스템

현재 공식은 XP와 반복 학습량이 레이팅을 직접 밀어 올리기 때문에 실력 판정용으로 부적합하다. ChartMon의 레이팅은 문제를 하나의 상대라고 보고, 유저 레이팅과 문제 난이도 레이팅의 차이로 기대 정답률을 계산하는 방식이 낫다.

### 문제 난이도 레이팅

각 문제는 `difficulty` 1-7을 가진다.

| difficulty | 문제 레이팅 `R_q` | 의미 |
|---:|---:|---|
| 1 | 850 | 입문 개념 확인 |
| 2 | 1000 | 기초 패턴/용어 적용 |
| 3 | 1150 | 일반 실전 차트 판독 |
| 4 | 1300 | 복합 근거 판단 |
| 5 | 1500 | 함정/노이즈 포함 |
| 6 | 1700 | 고난도 멀티타임프레임 |
| 7 | 1900 | 전문가급 실전 의사결정 |

### 단일 문제 업데이트

```text
E = 1 / (1 + 10 ^ ((R_q - R_u) / 400))
S = 정답이면 1, 오답이면 0
K = K_base * D * X * C
ΔR = K * (S - E)
R_new = R_u + ΔR
```

- `R_u`: 현재 유저 레이팅
- `R_q`: 문제 레이팅
- `E`: 현재 유저가 해당 문제를 맞힐 기대 확률
- `K_base`: 기본 32, 레이팅이 안정되면 16-24로 감소
- `D`: 난이도 가중치 `0.85 + 0.05 * difficulty`
- `X`: 반복 노출 감쇠 `1 / (1 + 0.35 * sameQuestionSeenCount)`
- `C`: 최근 30문제 일관성 계수 `0.9-1.1`

리뷰/복습으로 이미 푼 문제를 다시 맞히면 XP와 복습 진도에는 반영하되, `X` 때문에 레이팅 영향은 급격히 줄인다.

### 세션 업데이트

하루 15문제 데일리 트레이닝은 단일 문제 `ΔR`을 합산하되, 기대치 대비 성과를 세션 단위로 보정한다.

```text
sessionScore = Σ(S_i - E_i) / N
consistencyBonus = clamp(0.9, 1.1, 1 + sessionScore * 0.35)
ΔR_session = ΣΔR_i * consistencyBonus
```

즉 어려운 문제를 꾸준히 기대보다 잘 풀어야 올라가고, 쉬운 문제 반복은 거의 오르지 않는다.

### 티어 기준

초기에는 절대 레이팅 기준으로 운영하고, 유저 풀이 데이터가 5만-10만 답변 이상 쌓이면 백분위 보정 티어를 병행한다.

| 티어 | 레이팅 | 운영상 의미 | 데이터 누적 후 목표 백분위 |
|---|---:|---|---:|
| 입문 트레이더 | < 900 | 차트 구성요소와 용어 학습 단계 | 하위 20% |
| 기초 트레이더 | 900-1099 | 단일 패턴은 인식하지만 맥락 판단 약함 | 20-45% |
| 실전 초급 | 1100-1299 | 쉬운 실전 차트는 안정적으로 판독 | 45-65% |
| 중급 트레이더 | 1300-1499 | 복합 근거와 리스크 판단 가능 | 65-80% |
| 숙련 트레이더 | 1500-1699 | 노이즈/함정 문제에서도 기대 이상 | 80-92% |
| 시니어 트레이더 | 1700-1899 | 고난도 실전 의사결정 우수 | 92-97% |
| 엘리트 트레이더 | 1900+ | 상위권 고난도 문제에서 검증됨 | 상위 3% |

### 예시

유저 A의 현재 레이팅이 1200이고 difficulty 3 문제를 풀었다고 가정한다. difficulty 3의 `R_q`는 1150이다.

```text
E = 1 / (1 + 10 ^ ((1150 - 1200) / 400)) = 0.571
K = 32 * 1.00 * 1.00 * 1.00 = 32
정답: ΔR = 32 * (1 - 0.571) = +13.7
오답: ΔR = 32 * (0 - 0.571) = -18.3
```

자기 레이팅보다 쉬운 문제를 맞히면 적게 오르고, 틀리면 더 크게 내려간다. 반대로 1200 유저가 difficulty 5 문제(`R_q=1500`)를 맞히면 기대 확률이 낮기 때문에 크게 오른다.

## 2. 이론 분류 체계

현재 6개 대분류는 출발점으로는 괜찮지만, 실전 차트 교육 전체를 커버하기에는 부족하다. 특히 거래량/유동성, 멀티타임프레임, 실행/주문, 백테스트/복기, 시장 레짐이 빠져 있다. 권장 체계는 10개 대분류, 30개 중분류, 80개 소주제다.

### A. 차트와 시장 기초

- 차트 구성: OHLC, 캔들/바/라인 차트, 로그/선형 스케일
- 가격 형성: 호가, 스프레드, 체결, 슬리피지
- 시장 참여자: 개인/기관, 마켓메이커, 뉴스 이벤트

### B. 캔들스틱과 가격 행동

- 단일 캔들: 도지, 망치형, 장대양봉/음봉
- 복합 캔들: 장악형, 샛별/석별, 하라미
- 캔들 맥락: 위치, 거래량 결합, 타임프레임별 신뢰도

### C. 시장 구조

- 스윙 구조: HH/HL/LH/LL, 구조 전환
- 지지/저항: 수평 매물대, 역할 전환, 가짜 이탈
- 추세/채널: 추세선, 평행 채널, 추세 가속/둔화

### D. 패턴과 돌파

- 반전 패턴: 헤드앤숄더, 더블탑/바텀, 라운딩
- 지속 패턴: 플래그, 페넌트, 삼각수렴, 박스권
- 돌파 판단: 돌파-리테스트, 실패 돌파, 변동성 수축

### E. 거래량과 유동성

- 거래량 해석: 볼륨 스파이크, 감소 추세, 캔들-볼륨 불일치
- 유동성 구간: 스탑 밀집, 스윕, 이전 고저점
- 세션 특성: 장 시작/마감, 런던/뉴욕 세션, 저유동성 시간대

### F. 보조지표와 컨플루언스

- 추세 지표: 이동평균, MACD, ADX
- 모멘텀 지표: RSI, Stochastic, CCI
- 변동성/밴드: Bollinger Bands, ATR, Keltner Channel
- 다이버전스: 일반/히든, 실패 다이버전스, 과최적화 경계

### G. 멀티타임프레임과 시장 레짐

- 타임프레임 정렬: 상위 추세, 중간 구조, 하위 진입
- 레짐 분류: 추세장, 박스장, 고변동/저변동
- 시나리오 빌딩: 주/보조 시나리오, 무효화 조건

### H. 셋업과 진입/청산

- 셋업 정의: 진입 조건, 확인 조건, 금지 조건
- 주문 실행: 시장가/지정가/스탑, 분할 진입
- 청산 전략: 부분익절, 트레일링, 시간 기반 청산

### I. 리스크와 포지션 관리

- 손익비: R multiple, 기대값, 승률과 손익비 균형
- 포지션 사이징: 고정 금액, 고정 비율, 변동성 기반
- 계좌 리스크: 일간 손실 한도, 연속 손실 대응, 레버리지

### J. 심리, 루틴, 복기

- 심리 오류: FOMO, 복수매매, 확증편향
- 트레이딩 루틴: 체크리스트, 세션 준비, 뉴스 필터
- 복기/검증: 매매일지, 태그 분석, 백테스트와 포워드 테스트

## 3. 스킬 카테고리

레이더 차트는 모바일 가독성을 고려하면 8축이 최선이다. 내부 분석은 10개 이상으로 세분화하되, 홈 화면 레이더는 8축으로 압축한다.

| 축 | 정의 | 연결 이론 |
|---|---|---|
| 캔들/가격행동 | 캔들과 위치 맥락으로 단기 의도를 읽는 능력 | B |
| 시장구조/SR | 스윙, 지지저항, 구조 전환을 판정하는 능력 | C |
| 추세/레짐 | 추세장/박스장/변동성 환경을 구분하는 능력 | C, G |
| 패턴/돌파 | 차트 패턴과 돌파 실패/성공을 판단하는 능력 | D |
| 거래량/유동성 | 거래량, 스탑 구간, 유동성 스윕을 해석하는 능력 | E |
| 지표/컨플루언스 | 지표를 보조 근거로 합성하되 과신하지 않는 능력 | F |
| 셋업/실행 | 진입, 청산, 무효화 조건을 계획하는 능력 | H |
| 리스크/심리 | 손익비, 사이징, 감정 오류를 통제하는 능력 | I, J |

### 숙련도 계산

카테고리별로 단순 정확도 대신 카테고리 레이팅과 신뢰도 보정을 같이 쓴다.

```text
E_i = 1 / (1 + 10 ^ ((R_qi - R_ci) / 400))
K_i = 24 * difficultyWeight_i * novelty_i * recency_i
R_c_new = R_c + K_i * (S_i - E_i)

raw = clamp(0, 100, (R_c_new - 800) / 11)
N_eff = Σ(difficultyWeight_i * novelty_i * recency_i)
confidence = 1 - exp(-N_eff / 30)
trend = clamp(-8, 8, 20 * (EMA_recent_accuracy - EMA_long_accuracy))
proficiency = clamp(0, 100, 50 + confidence * (raw - 50) + trend)
```

- `R_c`: 해당 카테고리 전용 레이팅
- `N_eff`: 실질 풀이량. 어려운 신규 문제를 최근에 풀수록 크게 반영
- `confidence`: 표본이 적을 때 점수를 50 근처로 끌어오는 신뢰도
- `trend`: 최근 N문제 성과가 장기 평균보다 좋아지는지/나빠지는지

예를 들어 캔들 문제 3개를 모두 맞혀도 `confidence = 1 - exp(-3/30) = 0.095`라서 100점이 되지 않는다. 표본이 작으면 “좋은 출발”이지 “마스터”가 아니다.

### 문제 태깅 원칙

문제는 주 카테고리 1개만 가진다. 보조 태그는 여러 개를 허용하되 레이더 점수에는 주 카테고리만 반영한다. 그래야 MECE가 유지되고, “이 점수가 낮으면 어떤 이론을 공부해야 하는지”가 직접 연결된다.
